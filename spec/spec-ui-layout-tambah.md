# Tambah Pengisian BBM Page Layout Specification

![Status: Planned](https://img.shields.io/badge/status-Planned-blue)

Dokumen ini mendefinisikan layout untuk halaman **Tambah Pengisian BBM (tambah.html)** - form untuk menambah atau mengedit catatan pengisian BBM.

---

## 1. Requirements & Constraints

### Functional Requirements

- **REQ-FRM-001**: Form HARUS mendukung mode Add dan Edit (berdasarkan query param `?edit={id}`)
- **REQ-FRM-002**: Field wajib: Kendaraan, Tanggal, Liter, Total Harga, Odometer
- **REQ-FRM-003**: Field opsional: Jenis BBM, Catatan
- **REQ-FRM-004**: Harga per liter HARUS dihitung otomatis dan ditampilkan
- **REQ-FRM-005**: Form HARUS validasi sebelum submit
- **REQ-FRM-006**: Tanggal default = hari ini
- **REQ-FRM-007**: Warning jika odometer < odometer terakhir

### Validation Rules

- **VAL-001**: Liter: 0.01 - 999.99 (required)
- **VAL-002**: Total Harga: 1 - 99.999.999 (required)
- **VAL-003**: Odometer: 0 - 9.999.999 (required)
- **VAL-004**: Tanggal: tidak boleh di masa depan
- **VAL-005**: Catatan: maksimal 200 karakter

### Design Guidelines

- **GUD-FRM-001**: Mobile-first, single column layout
- **GUD-FRM-002**: Sticky submit button pada mobile
- **GUD-FRM-003**: Real-time validation feedback
- **GUD-FRM-004**: Clear focus states untuk accessibility
